向量搜索库 #

向量搜索库是一种专门用于存储、索引和查询高维向量的数据库，高维向量是特征或属性的数学表示。向量搜索库可以用于实现向量相似性搜索，即根据距离度量找出最接近的向量。向量搜索库可以应用于各种场景，如推荐系统、图像检索、语音识别、自然语言处理等。

目前，有几款比较流行的开源向量搜索库，如Faiss、Milvus、Proxima、Vearch、Jina等。它们各有特点和优势，具体可以参考以下链接：

  • Faiss：https://github.com/facebookresearch/faiss
  • Milvus：https://milvus.io/
  • Proxima：https://github.com/proximasearch/proxima
  • Vearch：https://github.com/vearch/vearch
  • Jina：https://github.com/jina-ai/jina

向量搜索库的原理是什么？ #

向量搜索库的原理是利用一种索引结构来存储和查询高维向量，高维向量是特征或属性的数学表示。向量搜索库的目的是找出与给定向量最相似的向量，即最近邻搜索。向量搜索库的核心问题是如何减少候选向量集和降低单个向量计算的复杂度。

一般来说，向量搜索库有两种索引结构：精确索引和近似索引。精确索引是指能够保证找到真正的最近邻的索引，如哈希表、树结构等；近似索引是指能够在牺牲一定精度的情况下，提高搜索效率的索引，如局部敏感哈希、乘积量化、倒排多索引等。

FAISS #

FAISS（Facebook AI Similarity Search）是一个开源库，针对高维空间中的海量数据，提供了高效且可靠的检索方法。它可以让开发者快速地搜索出相似的多媒体文档的嵌入向量。它解决了传统的基于哈希的搜索引擎的局限性，提供了更可扩展的相似度搜索功能。FAISS用C++编写，并提供与Numpy完美衔接的Python接口。FAISS支持十亿级别向量的搜索，是目前最为成熟的近似近邻搜索库。

简单来说，Faiss的工作，就是把我们自己的候选向量集封装成一个index数据库，它可以加速我们检索相似向量TopK的过程(https://zhuanlan.zhihu.com/p/357414033)

FAISS 的核心原理是什么？ #

FAISS 的核心原理其实就两个部分：Product Quantizer (PQ) 和 Inverted File System (IVF)。¹³

  • PQ 是一种向量量化方法，它将高维的向量分解成若干个低维的子向量，并对每个子向量进行聚类，得到一个码本（codebook）。然后，每个原始向量就可以用若干个码本中的编码来表示，从而实现数据压缩和降维。⁵⁶
  • IVF 是一种倒排索引结构，它将向量空间划分成若干个单元（cell），并将每个单元中的向量存储在一个列表中。然后，对于一个查询向量，只需要在它所属的单元和相邻的单元中进行搜索，从而减少搜索范围和时间。¹⁶

FAISS 还支持其他的向量检索算法，如 HNSW、LSH 等，但是它们都是基于 PQ 和 IVF 的改进或扩展。²⁴

FAISS有哪些应用场景？ #

FAISS有很多应用场景，主要是在需要对海量的高维向量进行相似度搜索或聚类的情况下。例如，在推荐系统中，可以用FAISS对用户和物品的嵌入向量进行快速匹配，从而实现个性化推荐；在人脸识别中，可以用FAISS对人脸特征向量进行实时检索，从而实现人脸识别；在自然语言处理中，可以用FAISS对文本或词语的嵌入向量进行相似度计算，从而实现文本分类、语义检索、文本生成等任务。

FAISS如何构建索引？ #

向量数据库 FAISS 是 Facebook 开源的一个高效的向量检索框架，它可以用于构建大规模的相似度搜索引擎。¹

要使用 FAISS 构建索引，首先需要选择一个合适的索引类型，比如 Flat、IVF、PQ 等，然后调用 index_factory 函数来创建索引对象。²

接下来，需要对索引进行训练和添加向量。训练是为了学习向量空间的分布，以便进行压缩和划分。添加向量是将原始的向量数据存储到索引中。³

训练和添加向量的代码示例如下：

python
import faiss
import numpy as np

# 生成一些随机向量
d = 64 # 向量维度
n = 10000 # 向量数量
xb = np.random.random((n, d)).astype('float32')

# 创建一个 IVF + PQ 的索引
index = faiss.index_factory(d, "IVF64,PQ8")

# 训练索引
index.train(xb)

# 添加向量
index.add(xb)

构建好索引后，就可以使用 search 或 search_and_reconstruct 方法来进行查询了。²

FAISS与其他相似度搜索库有什么区别？ #

FAISS 与其他相似度搜索库的区别主要有以下几点：

  • FAISS 是 Facebook AI 团队开源的一个专注于向量检索和聚类的库，它支持多种向量检索算法，如 Flat、IVF、PQ、HNSW 等，可以根据不同的需求选择合适的算法。¹²
  • FAISS 支持 CPU 和 GPU 计算，可以提高暴力搜索和 IVF 等算法的性能。¹⁴
  • FAISS 支持 Python 和 C++ 调用，可以与 Numpy 完美衔接。²⁵
  • FAISS 不支持动态更新索引，需要重新训练和添加向量。³⁶
  • FAISS 不支持自定义的相似度度量，只支持欧式距离和内积。³⁶
  • FAISS 不支持多模态的向量检索，如文本、图像、音频等。⁴⁶

其他相似度搜索库有：

  • Milvus 是一款开源的特征向量相似度搜索引擎，它集成了 FAISS、SPTAG 等开源库，并提供了更友好的 API 和界面，支持动态更新索引和多模态检索。⁴
  • ScaNN 是 Google 开源的一个高效的近似最近邻搜索库，它使用了一种新颖的数据压缩和分区技术，可以在保证高召回率的同时提高检索速度和降低内存占用。⁴
  • Jina 是一个基于神经网络的云原生搜索框架，它支持多种数据类型和模型，可以快速构建端到端的搜索应用。⁴

FAISS有什么优缺点？ #

FAISS 的优势和缺点主要取决于它的索引类型和参数设置。¹²³

一般来说，FAISS 的优势有：

  • 支持多种向量检索算法，如 Flat、IVF、PQ、HNSW 等，可以根据不同的需求选择合适的算法。
  • 支持数据压缩和降维，可以减少空间占用和加快检索速度。
  • 支持 GPU 加速，可以提高暴力搜索和 IVF 等算法的性能。
  • 支持分布式索引构建和查询，可以处理海量的向量数据。

FAISS 的缺点有：

  • 不支持所有的向量检索算法在 GPU 上运行，如 HNSW 等。
  • 不支持动态更新索引，需要重新训练和添加向量。
  • 不支持自定义的相似度度量，只支持欧式距离和内积。
  • 不支持多模态的向量检索，如文本、图像、音频等。

Annoy和Faiss有什么区别？ #

Annoy和Faiss的区别主要有以下几点¹：

  • Annoy不支持批量插入和查询，仅支持一种索引类型，单步查询速度快；Faiss支持批量插入和查询，支持100%召回、量化压缩等多种索引类型，并且能够使用GPU加速¹。

HNSW #

向量搜索库 HNSW 是一种基于图的近似最近邻搜索算法，它利用分层图提高图的导航性，减少无效计算，从而降低搜索时间¹。HNSW 的全称是 Hierarchical Navigable Small World，即“可导航小世界网络”²。它的基本思想是在不同的层次上构建不同密度的图，然后从高层向低层搜索，最终找到与查询向量最相似的目标向量⁴。HNSW 是一种高效且通用的向量索引算法，可以用于多种场景，如图像检索、文本检索、推荐系统等³。

Demo #

当然可以。Jina 提供了很多的例子和教程，你可以在 Jina 的 GitHub 仓库或者官网上找到。这里我给你展示一个简单的例子，用 Jina 实现一个基于文本的搜索引擎。首先，我们需要安装 Jina：

python
pip install jina

然后，我们需要准备一些文本数据，比如一些新闻标题。我们可以把它们存储在一个 txt 文件中，每行一个标题，比如：

text
Apple unveils new MacBook Air, MacBook Pro and Mac mini with its M1 chip
Tesla shares soar after crushing third-quarter earnings
Facebook launches cloud gaming service for Android and web users
Netflix raises prices on standard and premium plans
Amazon launches a rewards program for its independent ‘Flex’ drivers

接下来，我们需要定义一个 Executor，用来对文本进行向量化。我们可以使用 Jina Hub 中提供的 TransformerTorchEncoder，它可以利用预训练的 Transformer 模型将文本转换成向量。我们可以在一个 yaml 文件中配置这个 Executor，比如：

yaml
!TransformerTorchEncoder
with:
  pooling_strategy: auto
  pretrained_model_name_or_path: distilbert-base-cased
metas:
  name: encoder
requests:
  on:
    [IndexRequest, SearchRequest]:
      - !EncodeDriver {}

然后，我们需要定义一个 Flow，用来组织不同的 Executor。我们可以使用 Jina 的内置 Executor SimpleIndexer 来存储和检索向量，它基于 Numpy 实现了最近邻搜索。我们可以在另一个 yaml 文件中配置这个 Flow，比如：

yaml
!Flow
pods:
  encoder:
    uses: encoder.yml
    show_exc_info: true
    read_only: true
  indexer:
    uses: jinahub://SimpleIndexer/v0.4
    uses_metas:
      workspace: $JINA_WORKSPACE
    volumes: $JINA_WORKSPACE:/workspace/workspace
    polling: all
    timeout_ready: 600000
    needs: encoder

最后，我们需要写一个 Python 脚本来运行这个 Flow，并向它发送数据和查询。我们可以使用 Jina 的 Document 和 DocumentArray 类来封装数据，并使用 Client 类来与 Flow 交互。比如：

python
from jina import Document, DocumentArray, Flow, Client

# 创建一个 Flow 对象，并加载 yaml 配置文件
flow = Flow.load_config('flow.yml')

# 定义一些索引和查询的数据
index_data = DocumentArray.from_files('data.txt')
query_data = DocumentArray([Document(text='Apple releases new products')])

# 使用 with 语句启动 Flow，并等待它准备好
with flow:
  # 向 Flow 发送索引请求，将数据存储到 SimpleIndexer 中
  flow.index(inputs=index_data)
  
  # 向 Flow 发送搜索请求，根据查询返回最相似的数据
  results = flow.search(inputs=query_data, top_k=3)

# 打印搜索结果
for doc in results[0].docs:
  print(f'Query: {doc.text}')
  print('Top matches:')
  for match in doc.matches:
    print(f'- {match.text} (score: {match.scores["cosine"].value:.2f})')

运行这个脚本，我们就可以看到类似下面的输出：

Top matches:
- Apple unveils new MacBook Air, MacBook Pro and Mac mini with its M1 chip (score: 0.81)
- Amazon launches a rewards program for its independent ‘Flex’ drivers (score: 0.29)
- Tesla shares soar after crushing third-quarter earnings (score: 0.28)
